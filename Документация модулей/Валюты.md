# Документация к программе money\_conv()

### Назначение

Программа позволяет пользователю конвертировать денежные суммы между различными валютами. Все расчёты выполняются через промежуточное значение в долларах США (USD).

Поддерживаются следующие валюты:



Белорусский рубль (BYN)

Российский рубль (RUB)

Доллар США (USD)

Евро (EUR)

Фунт стерлингов (GBP)

Японская иена (JPY)

Канадский доллар (CAD)



### Зависимости

Стандартная библиотека C++ (<iostream>)

Пользовательский заголовочный файл: money\_conv.h

Обязательные функции, которые должны быть определены в реализации:

double to\_the\_usd(double currency\_rate, double amount) — конвертирует сумму из указанной валюты в USD, используя фиксированный курс

double serch\_res(double usd\_amount, double target\_rate) — конвертирует сумму в USD в целевую валюту



### Логика работы

Выбор исходной валюты

Пользователь выбирает валюту, из которой нужно выполнить конвертацию (опции 1–7).

Дополнительно (опция 8) можно вывести текущие курсы валют относительно USD.



Ввод суммы

Пользователь вводит числовое значение (money) — сумму в выбранной валюте.



Преобразование в USD

Сумма конвертируется в доллары США с помощью функции to\_the\_usd(currency\_rate, money), где currency\_rate — константа, соответствующая выбранной валюте.



Выбор целевой валюты

Пользователь выбирает валюту, в которую нужно перевести сумму (опции 1–7). Опция 8 (информация о курсе) в этом меню не приводит к расчёту результата.



Преобразование из USD в целевую валюту

Полученное значение в USD передаётся в функцию serch\_res(to\_usd, target\_rate), которая возвращает итоговую сумму в целевой валюте.



Вывод результата

Результат конвертации выводится с помощью cout.



Повтор выполнения

После завершения операции программа предлагает повторить конвертацию.

### Ограничения и замечания

Фиксированные курсы валют

Курсы заданы как константы и не обновляются динамически.



Нет проверки корректности ввода

Программа не проверяет, является ли введённое значение числом.



Использование double обеспечивает достаточную точность для учебных задач, но не рекомендуется для реальных финансовых расчётов (из-за ошибок округления с плавающей запятой). В профессиональных системах используют фиксированную точку (например, целые копейки/центы).
