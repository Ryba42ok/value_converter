Документация к программе num_syst()
 
Назначение
Программа позволяет пользователю преобразовывать целые положительные числа из одной системы счисления (СИ) в другую. Поддерживаются следующие системы счисления:

Двоичная (основание 2)
Восьмеричная (основание 8)
Десятичная (основание 10)
Шестнадцатеричная (основание 16)
Программа реализована как функция void num_syst() и предполагает использование внешней библиотеки numeral_system.h с объявлениями вспомогательных функций.

Зависимости
Стандартная библиотека C++ (<iostream>)
Пользовательский заголовочный файл: numeral_system.h
Обязательные функции, которые должны быть определены в реализации:
int search_first_non_zero(const char* str) — находит индекс первого ненулевого символа, однако, над данный момент отсутствует возможность отрицательного числа.
void toDec(const char* str, int* sum, int base, int index) — переводит число в десятичную СИ
Используется _itoa_s — нестандартная, но распространённая в MSVC функция преобразования целого числа в строку с указанием основания системы счисления.

Логика работы
Выбор исходной системы счисления
Пользователь выбирает, из какой СИ будет введено число (1–4).
Ввод числа как строки
Число считывается в символьный массив start_str (макс. длина — 100 символов).
Проверка корректности ввода
Программа проверяет каждый символ (начиная с первого ненулевого, с использованием search_first_non_zero) на соответствие цифрам допустимой СИ:
Двоичная: 0–1
Восьмеричная: 0–7
Десятичная: 0–9
Шестнадцатеричная: 0–9, A–F или a–f
Преобразование в десятичную систему
При корректном вводе строка преобразуется в десятичное число sum с помощью функции toDec.
Выбор целевой системы счисления
Пользователь выбирает, в какую СИ перевести число.
Преобразование и вывод результата
Число sum преобразуется в строку res_str с помощью _itoa_s и выводится посимвольно.
Повтор выполнения
После каждого преобразования программа предлагает повторить операцию.

Ограничения и замечания
Не поддерживает отрицательные числа.
Если входная строка содержит минус (-), он не обрабатывается корректно.
Регистронезависимость ввода
Шестнадцатеричные буквы принимаются как в верхнем (A–F), так и в нижнем (a–f) регистре.
